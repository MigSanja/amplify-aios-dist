# Second Brain (PARA + Inbox)

Dein Wissensspeicher. Struktur:
- **00_Inbox**: alles Neue landet hier, wird später einsortiert (`/sort-inbox`).
- **01_Identity**: wer du bist, Ziele, Stärken, Kommunikationsstil. `identity.md` liest Jarvis zuerst.
- **02_People**: Personen, Leads, Kunden (je eine Notiz, verlinkt).
- **03_Projects**: laufende Projekte mit Deadline/Ziel.
- **04_Areas**: Daueraufgaben ohne Enddatum (Bereiche/Verantwortungen).
- **05_Knowledge**: geprüftes Wissen, Playbooks, Learnings.

Regel: `brain/` speichert Wissen (wer/was/warum). Zahlen/KPIs liegen in `dashboard/data/`.
Secrets NIE hier, sondern in `.env`.

> Das Onboarding kopiert dieses Gerüst nach `brain/` bzw. bindet dein bestehendes Obsidian ein.
